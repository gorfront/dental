# Andreasyan Dental — API

Express + TypeScript backend for the Andreasyan Dental clinic management system.

## Stack
- **Runtime**: Node.js 20 + Express 4 + TypeScript
- **Database**: Neon PostgreSQL (via Drizzle ORM)
- **Auth**: Supabase Auth + JWT
- **Storage**: Supabase Storage (X-rays, avatars)
- **Deploy**: Render.com

## API Routes

| POST | `/api/auth/logout` | JWT | Logout |
| GET | `/api/auth/me` | JWT | Current user + profile |
| POST | `/api/auth/refresh` | — | Exchange Supabase token for JWT |
| GET | `/api/services` | — | List all services |
| POST | `/api/services` | admin | Create service |
| PUT | `/api/services/:id` | admin | Update service |
| GET | `/api/doctors` | JWT | List doctors |
| GET | `/api/doctors/:id/slots?date=` | JWT | Available time slots |
| GET | `/api/doctors/:id/appointments` | doctor/admin | Doctor's schedule |
| GET | `/api/appointments` | JWT | List appointments (role-filtered) |
| POST | `/api/appointments` | JWT | Book appointment |
| PATCH | `/api/appointments/:id/status` | doctor/admin | Update status |
| PATCH | `/api/appointments/:id` | doctor/admin | Reschedule |
| GET | `/api/patients` | doctor/admin | List patients |
| GET | `/api/patients/:id` | JWT | Patient details |
| GET | `/api/patients/:id/appointments` | JWT | Patient's appointments |
| GET | `/api/patients/:id/tooth-chart` | JWT | Tooth chart |
| PUT | `/api/patients/:id/tooth-chart/:tooth` | doctor/admin | Update tooth |
| GET | `/api/patients/:id/xrays` | JWT | List X-rays |
| GET | `/api/patients/:id/notes` | doctor/admin | Clinical notes |
| POST | `/api/patients/:id/notes` | doctor/admin | Add clinical note |

| GET | `/api/xrays/:id/url` | JWT | Refresh signed URL |

| GET | `/api/admin/schedule` | admin | Full schedule view |
| GET | `/api/admin/revenue` | admin | Revenue data |
| PATCH | `/api/admin/invoices/:id/pay` | admin | Mark invoice paid |
| GET | `/health` | — | Health check |

## Setup

```bash
# 1. Install
npm install

# 2. Environment
cp .env.example .env
# Fill in your values

# 3. Generate & run migrations
npm run db:generate
npm run db:migrate

# 4. Seed with mock data
npm run db:seed

# 5. Dev server
npm run dev
```

## Deploy to Render

1. Push to GitHub
2. Go to [render.com](https://render.com) → New → Web Service
3. Connect your GitHub repo
4. Render auto-detects `render.yaml`
5. Set environment variables in Render dashboard:
   - `DATABASE_URL`
   - `SUPABASE_URL`
   - `SUPABASE_SERVICE_ROLE_KEY`
   - `SUPABASE_PUBLISHABLE_KEY`
   - `JWT_SECRET` (auto-generated by Render)
   - `FRONTEND_URL` (your Runable site URL)
6. Deploy → your API will be live at `https://andreasyan-dental-api.onrender.com`

## Database Schema

```
users           → base auth table (links to Supabase Auth via supabase_id)
patients        → patient profile (1:1 with users)
doctors         → doctor profile (1:1 with users)
services        → dental service catalog
appointments    → bookings (patient + doctor + service)
tooth_chart     → per-tooth status per patient (FDI notation)
xrays           → X-ray metadata (files in Supabase Storage)
clinical_notes  → doctor notes per patient
invoices        → billing records
```
